## Q1 - 센서 성능 용어 [recall]
> 다음 두 상황에서 어떤 센서 성능 용어가 해당하는지 설명하라:
> (a) 같은 측정값에서 매번 다른 결과가 나온다.
> (b) 결과는 매번 같지만, 실제값과 다르다.

> [!answer]- 정답 보기
> (a) **정밀도(Precision) 낮음** — 반복성 부족, 노이즈/변동 큼
>
> (b) **정확도(Accuracy) 낮음, 정밀도 높음** — 체계적 오차(offset/bias) 존재
>    → 교정(calibration)으로 개선 가능

## Q2 - analogRead 변환 [recall]
> Arduino의 `analogRead(A0)`가 값 512를 반환했다. 공급 전압 5V에서 실제 입력 전압은?

> [!answer]- 정답 보기
> ADC 비율: 512 / 1023 = 0.500...
>
> 전압 = 0.5 × 5V = **약 2.5V**
>
> 더 정확히: (512 / 1023) × 5V = 2.503V

## Q3 - 리미트 스위치 연결 [application]
> 다리의 최대 올림 위치를 감지할 마이크로스위치를 Arduino 핀 2에 연결한다. `INPUT_PULLUP`을 사용할 때, 스위치가 눌렸을 때 핀 값은?

> [!answer]- 정답 보기
> **LOW (0)**
>
> `INPUT_PULLUP`: 내부에서 핀을 HIGH(5V)로 당김.
> 스위치 누름 → 핀이 GND에 연결 → **LOW**
>
> ```cpp
> pinMode(2, INPUT_PULLUP);
> if (digitalRead(2) == LOW) {  // 스위치 눌림
>   // 모터 정지
> }
> ```
>
> 반전 로직(눌림=LOW)에 주의!

## Q4 - 센서 선택 [application]
> 말 갤럽 중 캠샤프트가 1회전을 완료했는지 감지해야 한다. 저비용, 비접촉 방식의 최적 센서는?

> [!answer]- 정답 보기
> **홀 센서 (Hall Effect Sensor) + 자석**
>
> - 캠샤프트에 소형 자석 부착
> - 홀 센서 고정 (비접촉)
> - 자석이 지나갈 때 LOW 신호 → 1회전 감지
>
> 장점: 비접촉(마모 없음), 저비용, 빠른 응답
>
> 대안: 적외선 포토인터럽터, 마그네틱 엔코더

## Q5 - 신호 노이즈 [recall]
> 모터 구동 중 아날로그 센서 값이 불안정하게 튄다. 원인과 소프트웨어 해결책은?

> [!answer]- 정답 보기
> **원인:** 모터에서 발생하는 전자기 노이즈(EMI) → 센서 신호선에 유입
>
> **소프트웨어 해결 (이동 평균 필터):**
> ```cpp
> const int NUM_SAMPLES = 10;
> int readings[NUM_SAMPLES];
> int idx = 0;
>
> int smoothRead(int pin) {
>   readings[idx] = analogRead(pin);
>   idx = (idx + 1) % NUM_SAMPLES;
>   int sum = 0;
>   for (int i = 0; i < NUM_SAMPLES; i++) sum += readings[i];
>   return sum / NUM_SAMPLES;
> }
> ```
>
> **하드웨어 해결:** 100nF 세라믹 커패시터를 센서 VCC-GND에 추가

## Q6 - I2C 주소 스캔 [application]
> 새 I2C 센서를 연결했는데 주소를 모른다. 어떻게 확인하는가?

> [!answer]- 정답 보기
> **I2C 스캐너 코드 실행:**
>
> ```cpp
> #include <Wire.h>
> void setup() {
>   Wire.begin();
>   Serial.begin(9600);
>   for (byte addr = 1; addr < 127; addr++) {
>     Wire.beginTransmission(addr);
>     if (Wire.endTransmission() == 0) {
>       Serial.print("발견: 0x");
>       Serial.println(addr, HEX);
>     }
>   }
> }
> void loop() {}
> ```
>
> → 시리얼 모니터에서 장치 주소 확인
> 예: MPU-6050 → 0x68, PCA9685 → 0x40

## Q7 - MPU-6050 활용 [application]
> 라이더 상체의 기울기를 MPU-6050으로 감지해 자세를 피드백 제어하려 한다. 어떤 데이터를 사용하고 어떻게 연결하는가?

> [!answer]- 정답 보기
> **사용 데이터:** 가속도계 (X, Y, Z축) → 기울기 각도 계산
> (자이로스코프는 각속도 → 드리프트 있음. 두 가지를 상보 필터로 합산하면 정밀)
>
> **연결:**
> - SDA → Arduino A4
> - SCL → Arduino A5
> - VCC → 3.3V (⚠️ 5V 연결 시 손상! 레벨 시프터 사용)
> - GND → GND
>
> **라이브러리:** `MPU6050` (Wire.h 포함)
> I2C 주소: 0x68 (AD0=GND), 0x69 (AD0=VCC)

## Q8 - 센서 vs 오픈루프 [analysis]
> 말 갤럽 제어에서 센서 없이 서보 각도만으로 제어하는 오픈루프와 리미트 스위치로 위치를 확인하는 방식의 장단점을 비교하라.

> [!answer]- 정답 보기
> **오픈루프 (센서 없음):**
> - 장점: 단순, 저비용, 코드 쉬움
> - 단점: 서보 과부하 시 위치 정확도 저하, 안전장치 없음
>
> **리미트 스위치 폐루프:**
> - 장점: 물리적 한계 초과 방지 (안전), 초기화(homing) 가능
> - 단점: 스위치 배선 추가, 코드 복잡
>
> **권장:** 전시 작품은 리미트 스위치 필수 — 관람객 환경에서 예상치 못한 충돌 발생 가능
