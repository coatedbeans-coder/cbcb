## 프로젝트에 필요한 센서

### 리미트 스위치 (마이크로스위치)

> [!important] 안전 장치 필수 센서
> 캠/링크가 물리적 한계 초과 시 모터 즉시 정지.

```cpp
// 리미트 스위치 연결
const int LIMIT_PIN = 2;  // 인터럽트 핀

void setup() {
  pinMode(LIMIT_PIN, INPUT_PULLUP);
  attachInterrupt(digitalPinToInterrupt(LIMIT_PIN),
                  limitISR, FALLING);
}

volatile bool limitTriggered = false;
void limitISR() {
  limitTriggered = true;
}
```

### 로터리 엔코더

모터/서보 각도 측정. 폐루프 제어에 활용.

```cpp
volatile long encoderCount = 0;

void encoderISR() {
  int A = digitalRead(ENC_A);
  int B = digitalRead(ENC_B);
  if (A == B) encoderCount++;
  else encoderCount--;
}
```

### IMU (가속도계 + 자이로)

라이더 상체 기울기 감지 → 자세 피드백 제어.

- **MPU-6050**: I2C, 6축 (가속도 3축 + 자이로 3축)
- I2C 주소: 0x68

## 센서 성능 용어 (Bolton Mechatronics)

| 용어 | 설명 |
|------|------|
| **정밀도 (Precision)** | 반복 측정의 일관성 |
| **정확도 (Accuracy)** | 실제값과의 차이 |
| **분해능 (Resolution)** | 감지 가능한 최소 변화량 |
| **선형성 (Linearity)** | 입출력 관계의 직선성 |
| **응답 시간** | 변화 감지까지의 지연 |
| **드리프트 (Drift)** | 시간에 따른 오차 변화 |

## 아날로그 vs 디지털 센서

```
아날로그 센서:
  → analogRead() → 0~1023 → 물리량으로 변환
  예: 포텐시오미터, FSR, LDR

디지털 센서:
  → I2C/SPI 통신 또는 단순 HIGH/LOW
  예: 마이크로스위치, 초음파(HC-SR04), DHT22

온/오프 센서 (스위치형):
  → digitalRead() → HIGH/LOW
  예: 리미트 스위치, 버튼
```

## 센서 선택 기준 (Bolton)

```
1. 측정 범위: 예상 입력값 포함?
2. 분해능: 필요 정밀도 충족?
3. 응답 시간: 시스템 속도와 맞음?
4. 환경 조건: 온도, 진동, 습도 적합?
5. 인터페이스: Arduino와 호환?
6. 비용: 예산 내?
```

## Exam/Test Patterns

| 시나리오 / 키워드 | 답 |
|-------------------|----|
| "과구동 방지, 물리적 한계 감지" | 마이크로스위치 (리미트 스위치) |
| "MPU-6050 인터페이스" | I2C (주소 0x68) |
| "측정 반복성의 지표" | 정밀도 (Precision) |
| "아날로그 센서 → Arduino" | `analogRead()`, 0~1023 |
| "가장 저렴한 각도 센서" | 포텐시오미터 |
