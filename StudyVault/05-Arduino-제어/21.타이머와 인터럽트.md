## millis() 기반 논블로킹 패턴

> [!important] 핵심 패턴 — 반드시 암기
>
> ```cpp
> unsigned long previousMillis = 0;
> const long interval = 1000;  // 1초 간격
>
> void loop() {
>   unsigned long currentMillis = millis();
>
>   if (currentMillis - previousMillis >= interval) {
>     previousMillis = currentMillis;  // 타이머 갱신
>     // 여기서 작업 실행
>     doSomething();
>   }
>   
>   // 다른 작업도 동시에 실행 가능!
>   doOtherThings();
> }
> ```
>
> **핵심 원리**: 빼기 방식으로 비교 → millis() 오버플로우 시에도 안전

## 다중 타이머 패턴 (갤럽 제어)

```cpp
// 각 다리마다 독립 타이머
unsigned long legTimer[4]   = {0, 250, 500, 750}; // 오프셋 초기화
int           legState[4]   = {0, 0, 0, 0};
const long    LEG_PERIOD    = 1000; // 갤럽 주기 1초
const long    LIFT_DURATION = 250;  // 들기 시간

void loop() {
  unsigned long now = millis();

  for (int i = 0; i < 4; i++) {
    unsigned long elapsed = now - legTimer[i];

    if (legState[i] == 0 && elapsed >= LEG_PERIOD) {
      // 다리 들기 시작
      servos[i].write(UP_ANGLE);
      legTimer[i] = now;
      legState[i] = 1;
    }
    else if (legState[i] == 1 && elapsed >= LIFT_DURATION) {
      // 착지
      servos[i].write(DOWN_ANGLE);
      legTimer[i] = now;
      legState[i] = 0;
    }
  }
}
```

## 인터럽트 (Interrupt)

### 외부 인터럽트 (External Interrupt)

특정 핀의 상태 변화 시 ISR(인터럽트 서비스 루틴) 즉시 실행.

```cpp
volatile bool buttonPressed = false;

void ISR_button() {
  buttonPressed = true;  // ISR 내에서는 최소 작업만!
}

void setup() {
  attachInterrupt(
    digitalPinToInterrupt(2), // 핀 2 (Uno: 2, 3번만 지원)
    ISR_button,               // ISR 함수
    FALLING                   // 트리거: RISING, FALLING, CHANGE
  );
}

void loop() {
  if (buttonPressed) {
    buttonPressed = false;
    // 실제 처리는 loop()에서
    handleButton();
  }
}
```

> [!warning] ISR 규칙
> - ISR 내에서 `delay()` 사용 금지
> - ISR 내에서 `Serial.print()` 사용 금지
> - ISR에서 수정하는 변수는 반드시 `volatile` 선언
> - ISR은 최대한 짧게 (플래그 세우고 loop에서 처리)

**인터럽트 지원 핀 (Arduino Uno):**
- 핀 2 (Interrupt 0)
- 핀 3 (Interrupt 1)

## millis() 오버플로우 처리

```
millis()는 unsigned long (32비트)
최대값: 4,294,967,295 ms ≈ 49.7일
→ 49.7일 후 0으로 리셋

안전한 비교 방법:
  if (millis() - lastTime >= interval)  ← ✅ 안전
  if (millis() >= lastTime + interval)  ← ❌ 오버플로우 취약
```

## 하드웨어 타이머 (고급)

Arduino Uno에는 3개 타이머: Timer0, Timer1, Timer2.

```
Timer0: 8비트, delay()/millis()/micros() 사용 중 → 건드리면 delay 망가짐
Timer1: 16비트, 고정밀 타이밍에 사용 가능
Timer2: 8비트, tone() 사용 중
```

**TimerOne 라이브러리 사용 (정밀 주기 인터럽트):**
```cpp
#include <TimerOne.h>

void timerISR() {
  // 1ms마다 실행
  updateMotors();
}

void setup() {
  Timer1.initialize(1000);    // 1000 us = 1ms
  Timer1.attachInterrupt(timerISR);
}
```

## 갤럽 타이밍 설계

```
갤럽 1 Hz (1초/주기):
  전체 주기 = 1000ms
  각 다리 오프셋 (올바른 갤럽 순서: 우뒤→좌뒤→우앞→좌앞):
    우뒤: 0ms
    좌뒤: 250ms
    우앞: 500ms
    좌앞: 750ms

  1다리 타임라인 (1000ms 기준):
    0ms ~ 100ms:   상승 (다리 들기)
    100ms ~ 200ms: 드웰 (공중)
    200ms ~ 350ms: 하강 (착지)
    350ms ~ 999ms: 드웰 (착지 유지)
```
