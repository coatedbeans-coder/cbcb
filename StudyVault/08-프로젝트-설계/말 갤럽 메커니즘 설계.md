# 말 갤럽 메커니즘 설계 (★★★)

#project #mechanism #cam

## 말의 갤럽 보행 분석

### 실제 말 갤럽 순서

```
갤럽 (4박자 순서): 우뒤 → 좌뒤 → 우앞 → 좌앞

시간 →   0%    25%   50%   75%   100%
우뒤다리  ▐████░░░░░░░░▐████░░░░░░░░
좌뒤다리  ░░░░▐████░░░░░░░░▐████░░░░
우앞다리  ░░░░░░░░▐████░░░░░░░░▐████
좌앞다리  ░░░░░░░░░░░░▐████░░░░░░░░░

▐████ = 공중 (다리 올림)
░░░░ = 착지 (다리 내림)
```

**핵심**: 각 다리가 1/4 주기(90°/25%) 씩 위상 차이. 뒷다리가 먼저 박차고 앞다리가 착지.

## 설계 A: 캠샤프트 방식

```
1개 기어드 DC 모터 → 캠샤프트 → 4개 캠 → 4개 다리

캠샤프트 (갤럽 순서: 우뒤→좌뒤→우앞→좌앞):
  ├── 캠1 (0°)   → 우뒤다리 로드
  ├── 캠2 (90°)  → 좌뒤다리 로드
  ├── 캠3 (180°) → 우앞다리 로드
  └── 캠4 (270°) → 좌앞다리 로드
```

**캠 설계 (1개 기준):**
```
캠 반지름: 최소 = 30mm, 최대 = 50mm
스트로크 = 50 - 30 = 20mm

캠 프로파일 (각도 기준):
  0°~90°:    상승 (sinusoidal rise) → 다리 들기  [리프트 90°]
  90°~120°:  드웰 (최고점 유지) → 공중           [공중 드웰 30°]
  120°~270°: 하강 (sinusoidal fall) → 착지        [착지 150°]
  270°~360°: 드웰 (최저점 유지) → 착지 유지       [착지 드웰 90°]

※ 비대칭 의도: 말은 다리를 빠르게 들고(90°) 느리게 착지(150°).
  착지 드웰(90°)이 공중 드웰(30°)보다 3배 긴 것은 의도된 설계 —
  실제 갤럽에서 다리가 지면에 닿는 시간이 공중 시간보다 길다.
```

**링크장치:**
```
캠 팔로워 → 커넥팅 로드 → 다리 로커
  
  ↕ (수직 왕복)       (회전 변환)    (각도 스윙)
  20mm 스트로크  →  100mm 로드  →  ±15° 스윙
```

## 설계 B: 서보 방식 (권장)

```
Arduino + PCA9685
  │
  ├── 서보1 (채널0) → 우앞다리 크랭크-링크
  ├── 서보2 (채널1) → 우뒤다리 크랭크-링크
  ├── 서보3 (채널2) → 좌앞다리 크랭크-링크
  └── 서보4 (채널3) → 좌뒤다리 크랭크-링크
```

**각 다리 서보 각도 계획:**
```
착지 위치: 0° (서보 0도)
공중 위치: 45° (서보 45도)

갤럽 1주기 = 1000ms

우뒤 타임라인 (기준, 0ms 시작):
  0ms:       0° 상승 시작 (착지 위치에서 바로 들기 시작)
  0~100ms:   0→45° 상승
  100~200ms: 45° 유지
  200~350ms: 45→0° 하강
  350~1000ms: 0° 유지 (착지)

좌뒤 타임라인: 위와 동일, 250ms 오프셋
우앞 타임라인: 위와 동일, 500ms 오프셋
좌앞 타임라인: 위와 동일, 750ms 오프셋
```

## 다리 링크 기하학

```
서보 혼(크랭크) → 커넥팅 로드 → 다리 섹션

            서보 혼
              ●──────────────────● 다리 상부 피봇 (어깨)
              │ 크랭크 r=35mm    │
              │커넥팅로드 L=100mm│
              ●──────────────────● 다리 접지부 (발굽)
              다리 길이 D = 150mm (어깨 피봇 ~ 발굽)

서보 45° 회전 시 다리 스윙 각도:
  수평 변위 x = r × sin(45°) = 35 × 0.707 = 24.7mm
  스윙 각도 θ = arcsin(x / D) = arcsin(24.7 / 150)
              = arcsin(0.165) ≈ 10°

→ 자연스러운 갤럽 스윙 ✓
※ r이 작으면 스윙 부족. D=150mm 기준 r=35mm로 ~10° 확보.
```

## 말 고개 동작 연동

```
캠샤프트 방식 시:
  캠5 (45°) → 고개 끄덕임 로드
  → 갤럽 1회당 고개 2회 끄덕임

서보 방식 시:
  서보5 (채널4) → 목 관절
  → 갤럽 주기의 1/2 주파수로 끄덕임
```

## 제작 순서

```
1. 다리 링크 치수 계산 (Grashof 검증)
2. 합판으로 다리 형상 레이저 컷
3. 피봇(M5 볼트 + 볼 베어링) 조립
4. 서보 or 캠 마운팅
5. 단일 다리 동작 테스트
6. 4다리 동기화 테스트
7. 타이밍 미세 조정
```

## 트러블슈팅 가이드

| 증상 | 원인 | 해결 |
|------|------|------|
| 다리가 갑자기 멈춤 | 서보 과부하 | 더 큰 서보 or 링크 길이 조정 |
| 갤럽이 어색함 | 타이밍 오프셋 틀림 | 위상 오프셋 값 조정 |
| 진동 심함 | 질량 불균형 | 무게 배분 재설계 |
| 링크 잠김(Lock-up) | Grashof 조건 미충족 | 링크 길이 재설계 |
